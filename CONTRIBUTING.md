# Contributing to JADMan

First off, thank you for considering contributing to JADMan! It's people like you that make JADMan such a great tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for JADMan. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Use a clear and descriptive title** for the issue to identify the problem.
- **Describe the exact steps which reproduce the problem** in as many details as possible.
- **Explain which behavior you expected to see and why.**
- **Provide specific examples to demonstrate the steps.** Include links to files or copy-pasteable snippets.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for JADMan, including completely new features and minor improvements to existing functionality.

- **Use a clear and descriptive title** for the issue to identify the suggestion.
- **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
- **Describe the current behavior and explain which behavior you expected to see instead** and why.
- **Explain why this enhancement would be useful** to most JADMan users.

### Pull Requests
- Fill in the pull request template.
- Ensure the test suite passes (`cargo test`).
- Follow the existing code style (run `cargo fmt`).
- Ensure no new Clippy warnings are introduced.

## Styleguides

### Rust Styleguide
All Rust code is formatted with `rustfmt`. You can run `cargo fmt` to format your code.
